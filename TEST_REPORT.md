## 真实操作流程测试（23 项） — 模拟用户完整操作

| 流程 | 测试数 | 模拟的操作 |
|------|--------|-----------|
| 1. 灵感输入 → AI 规划 | 5 | `textarea.fill()` 输入灵感 → 点击建议按钮 → 设置风格/字数 → 点击 AI 规划 → 等待响应 |
| 2. 编辑器操作与持久化 | 3 | `快速开始` 进入编辑器 → 写入内容 → localStorage 持久化验证 → 蓝图联动 |

| 5. 端到端完整写作流程 | 3 | 创建文章 → AI 规划(蓝图) → 写作 → 保存 → 刷新后数据仍在 → 字数统计 |
| 6. 文章与集合管理 | 3 | 新建文档按钮 → 侧边栏展示 → 系列规划创建 |
| 7. 错误处理与边界 | 3 | 空灵感(按钮禁用) → 特殊字符输入 → 快速连续点击(不崩溃) |

### 具体验证点举例

```
流程 1.1:  page.locator(".startup-splash__input").fill("人工智能对未来教育的影响和变革")
            → 验证输入框值 → 验证 AI 规划按钮已启用

流程 1.2:  page.locator("button", { hasText: "写一篇关于秋天午后的散文" }).click()
            → 验证灵感文本已填入输入框

流程 2.1:  page.locator("button", { hasText: "快速开始" }).click()
            → page.locator('[contenteditable="true"]').fill("测试文章内容")

流程 4.1:  page.locator(".sidebar__nav button", { hasText: "设置" }).click()
            → page.waitForSelector(".settings-modal") 验证面板打开

流程 5.1:  fill("远程工作的未来趋势") → click("AI 规划") 
            → localStorage.setItem(蓝图+内容) → 验证三者数据一致性
```
