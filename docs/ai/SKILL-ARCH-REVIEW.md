# 技能架构评估与重构方案

## 现状分析

目前存在两套平行的"技能"系统：

### 系统 A：快捷技能（`src/lib/skill.ts`）
- **定位**：选中文本后的即时 AI 操作（润色/改写/翻译等）
- **使用场景**：InlineToolbar（浮动工具栏）、AgentPanel（AI 对话面板）
- **数量**：18 个内置技能
- **状态**：硬编码列表，无管理 UI，无自定义能力

### 系统 B：写作风格（`src/lib/writingSkill.ts`）
- **定位**：从灵感→标题→简介→大纲→标签→正文的全流程写作控制
- **使用场景**：StartupSplash（写作入口选择）、plan.ts（分步生成）
- **数量**：8 个内置风格
- **状态**：有管理 UI（Settings → 写作风格），支持自定义

### 核心问题

1. **功能重叠**：两套系统都包含 "academic/学术"、"creative/创意"、"blog/博客" 等，但含义不同、互不相通
2. **快捷技能未纳入管理**：用户无法在设置中查看/编辑/关闭快捷技能
3. **设置 Tab 混淆**：同时存在 "技能" 和 "写作风格" 两个 tab，用户分不清区别
4. **大量技能与编辑器写作无关**：citation（引用格式）、readability（可读性）、keyword-extract（关键词提取）等并非日常写作操作

## 重构方案

### 核心理念：两个系统，清晰分工

```
┌─────────────────────────────────────────────────┐
│                 写作风格                          │
│  (Writing Styles - 全流程控制)                    │
│                                                   │
│  灵感 → 标题 → 简介 → 大纲 → 标签 → 正文         │
│  StartupSplash 入口选择                           │
│  Settings 中管理/自定义                           │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│                 快捷技能                          │
│  (Quick Skills - 选区即时操作)                    │
│                                                   │
│  选中文本 → 浮动工具栏 / AI 面板                  │
│  润色 / 改写 / 翻译 / 扩写 / 续写 / 分析          │
│  轻量级，一段 prompt 即可                         │
│  Settings 中可开关                                │
└─────────────────────────────────────────────────┘
```

### 写作风格（已有，持续完善）
- 9+ 个内置风格（通用/学术/博客/创意/自媒体/技术教程/商业文案/新闻报道/营销文案/产品文档/书评影评）
- 自定义风格继续完善
- StartupSplash 下拉选择 + AI 工具栏快速切换

### 快捷技能（需要改造）

#### 保留哪些技能

与"选中文本后即时操作"强相关的才保留：

| 技能 | 说明 | 保留 |
|------|------|------|
| polish | 润色文本，使语言流畅自然 | ✅ |
| rewrite | 改写选中文本，提升表达质量 | ✅ |
| translate | 翻译文本 | ✅ |
| expand | 扩写段落，补充论据细节 | ✅ |
| continue-writing | 从光标位置继续写作 | ✅ |
| proofread | 语法校对、错别字检查 | ✅ |
| paraphrase | 同义改写，保留原意改变句式 | ✅ |
| summary | 生成摘要 | ✅ |

#### 移除的技能

| 技能 | 原因 |
|------|------|
| academic | 与写作风格「学术严谨」重叠 |
| creative | 与写作风格「创意文学」重叠 |
| blog | 与写作风格「博客口语」重叠 |
| novel | 与写作风格「创意文学」重叠 |
| headline | 标题生成由写作风格流程覆盖 |
| email | 与写作风格「商业文案」重叠 |
| outline | 大纲生成由写作风格流程覆盖 |
| keyword-extract | 非日常编辑操作 |
| readability | 非日常编辑操作 |
| citation | 非日常编辑操作 |

#### 可开关管理

快捷技能在 Settings 中增加管理界面：
- 列表展示所有快捷技能（名称 + 描述 + 图标）
- 每个技能有开关 toggle
- 关闭的技能不在 InlineToolbar 和 AgentPanel 中显示

#### 自定义提示词（未来扩展）

快捷技能本质是"一段提示词 + 参数"。后续可开放：
- 创建自定义快捷技能（指定名称、图标、prompt body）
- 编辑内置技能的 prompt

## 实施步骤

### 第一步：清理快捷技能列表
- 从 `getFallbackSkills()` 移除冗余技能
- 仅保留 8 个核心技能

### 第二步：Settings 增加快捷技能管理
- 在设置中新增 "快捷技能" 管理区域（放在 写作风格 tab 下方 或 独立区域）
- 每个技能可开关
- 保存到 localStorage（浏览器模式）/ Tauri store（桌面模式）

### 第三步：打通开关状态
- InlineToolbar 和 AgentPanel 读取启用状态
- 禁用的技能不在 UI 中显示

### 第四步：合并设置 Tab（争议选项）
- 移除独立的 "技能" tab（原 Zap 图标）
- 将快捷技能管理合并到 "写作风格" tab 中
- 或保留独立 tab 但改名更清晰

## 实施结果（v1.4.0 回顾）

### 已执行的事项
- **快捷技能保留 8 个核心**：polish、rewrite、translate、expand、continue-writing、proofread、paraphrase、summary（与方案一致）
- **写作风格统一管理**：9+ 种内置风格（通用/学术/博客/创意/自媒体/技术教程/商业文案/新闻报道/营销文案/产品文档/书评影评），通过 StartupSplash 下拉选择 + AI 工具栏快速切换
- **风格快速切换**：AI 工具栏增加写作风格快速切换下拉菜单
- **技能维度进度条**：技能卡片显示维度进度条可视化

### 未执行 / 后续考虑
- 设置 Tab 未合并（"技能" 和 "写作风格" 仍独立存在）
- 快捷技能管理 UI 未在设置中补充（仍在 Settings > 写作风格 中统一管理）
- 内置技能的 prompt 编辑暂未开放

### 遗留建议
- 后续可考虑合并设置 Tab，降低用户混淆
- 快捷技能的自定义 prompt 能力可在需求明确后补充
