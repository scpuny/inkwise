## 二、当前 UX 问题清单

### 🔴 P0 问题

#### 1. 三层面板同时打开时视觉过载

侧栏 + 编辑器 + 样式面板 + AI 面板可以同时打开，对于新用户来说 3-4 个并列区域信息量太大。

```
当前场景：
  StylePanel(右) + AgentPanel(右) + Sidebar(左) + Editor(中)

问题：右侧两个面板（StylePanel + AgentPanel）都是右置，
      它们之间没有切换关系，可以同时打开，压缩编辑器空间。
```

**改法**：StylePanel 并入 AI 面板的 tab 中，或作为 AI 面板内的一个二级抽屉。

#### 2. 焦点模式退出按钮不醒目

用户进入焦点模式后，退出按钮在屏幕正中下方，小字 "Exit Focus"，不符合中文用户习惯。

```
当前:
  [Exit Focus]  ← 纯英文、位置突兀、容易被忽略

改法:
  右上角或 Esc 提示 "退出焦点模式" 中文按钮
  或用 Escape 退出（已有快捷键但无视觉提示）
```

#### 3. 新建文档流程有感知延迟

```
点击"新建文档"
  → onNewArticle()
  → setDocPickerOpen(true)     ← 弹出选择合集的弹窗
  → 用户选择合集
  → addArticle() + saveArticleContent()
  → EditorPane 加载

问题：新建文档需要先选择合集，增加了步骤。
      大多数用户的直觉是"点新建 → 直接进编辑器写"。
```

**改法**：新建文档默认放到最近使用的合集或"默认合集"，用户可在编辑器内通过 `ArticleHeader` 移动到其他合集。

#### 4. 保存状态反馈不直观

StatusBar 的保存状态展示为一段文字"保存中…"/"已保存"，且位于屏幕最底部。用户视觉焦点的自然落点在编辑器区域，不会主动看底部。

```
当前:
  保存状态在 StatusBar 最底部

改法:
  编辑器右上角显示保存状态指示器（绿点/旋转图标），
  StatusBar 保留但作为辅助信息
```

### 🟡 P1 问题

#### 5. Toolbar 缺少常用格式快捷键提示

当前的 Toolbar 有 Bold/Italic/Underline 等按钮，但 hover 时不显示键盘快捷键，新用户需要记忆或摸索。

```
当前:   Toolbar 按钮只有 title="粗体" / title="斜体"
期望:   hover 时显示 "粗体 (⌘B)" / "斜体 (⌘I)"
```

#### 6. AI 面板三个 tab 利用率低

AgentPanel 有 "对话" 和 "评估" 两个 tab，"历史" 标签被注释掉了。但 "评估" 功能需要用户主动切换到该 tab 才能看到，不符合直觉。

```
当前: 审阅结果在 AI 面板的 "评估" tab
        用户写完文章 → 点击审阅 → AI 面板自动弹出
        但切到 "评估" tab 这个动作不是自动的

改法: 审阅完成后自动切换到 "评估" tab 并在面板标题闪烁提示
```

#### 7. 文章标题编辑不够直观

当前文章标题在 `ArticleHeader` 中显示，但点击标题不会进入编辑模式，需要打开蓝图编辑器（弹窗）才能修改标题。

```
用户期望: 双击标题直接编辑
当前: 需要点击蓝图编辑器图标 → 弹窗 → 在弹窗中修改标题
```

#### 8. 大纲和编辑器之间缺少联动

大纲面板在 Sidebar 中，点击大纲项应该滚动编辑器到对应章节。当前有 `outline-navigate` 事件但用户感知不强。

```
改进: 
  - 点击大纲项时编辑器平滑滚动到对应标题
  - 编辑器滚动时大纲面板高亮当前章节
  - 编辑器内的标题变更自动同步到大纲（当前已 partial 实现）
```

#### 9. 写作阶段切换缺少引导

用户写完文章后，从 "writing" 切换到 "reviewing" 阶段时，没有任何引导提示 "是否需要先生成 AI 审阅"。

```
改进: 阶段切换时弹出确认/引导对话框：
  "切换到审阅阶段？建议先保存当前内容并生成 AI 审阅报告"
  并提供"立即审阅"和"稍后再说"两个选项
```

#### 10. AIBar（底部指令条）默认展开高度过大

AIBar 的 `COMPOSER_MIN_HEIGHT = 104`，即使没有文字也占据 104px 底部空间，在焦点模式下浪费屏幕。

```
改进: 无输入时折叠为单行 36px 高度，聚焦或输入时展开
```

#### 11. 项目面板和编辑器互斥切换突兀

当有关联项目时，`projectPanelOpen && !hasActiveArticle` 判定在编辑器位置显示 ProjectExplorer。进入文章后变成编辑器，退出文章后变回 ProjectExplorer。

```
问题: 用户在项目面板查看项目 → 点击一篇文章 → 面板变编辑器
      关闭文章后回到项目面板（状态保持）
      但如果用户想在写文章的同时查看项目，做不到

改进: 项目面板应作为侧栏的一个 tab，而非替代编辑器
```

### 🟢 P2 问题

#### 12. 状态栏信息过多

StatusBar 显示：字数、字符数、段落数、阅读时间、保存状态、阶段、模型、effort、项目分析状态、空闲/就绪 共 10+ 项信息。

```
改进: 默认只显示字数 + 保存状态 + 模型
      其余信息通过点击展开或设置中配置
```

#### 13. 暂无文章时的空白态

当前 "无文章" 时显示 `StartupSplash` 组件。内容为 Logo + 欢迎语 + 快捷操作。缺少引导新用户的第一步提示。

```
改进: StartupSplash 增加：
  - "快速开始：选择一个合集开始写作" 的引导步骤
  - 示例文章入口（第一篇文章的引导）
  - 快捷键提示
```

## 三、布局调整方案

### 核心原则

1. **保持主题系统不变**（theme.ts, articleThemes.ts）
2. **只调整界面布局和交互逻辑**
3. **减少默认展示的面板数**
4. **常用操作减少点击次数**
5. **不常用的功能折叠或合并**

### 新布局

```
┌──────────────────────────────────────────────────────────┐
│  Sidebar (240-300px)  │        Editor (1fr)             │
│  ┌────────────────┐   │  ┌───────────────────────────┐  │
│  │ 品牌/Logo      │   │  │ [保存状态] [阶段] [更多▾] │  │  ← 顶部操作栏
│  │ 搜索框         │   │  ├───────────────────────────┤  │
│  │ ──────────     │   │  │ 标题（双击编辑）           │  │
│  │ tab: [目录|大纲| │  │  │ Toolbar (紧凑，带快捷键)    │  │
│  │       项目]   │   │  │ EditorContent (TipTap)     │  │
│  │ ──────────     │   │  │                           │  │
│  │ 内容区         │   │  │ InlineGhostText            │  │
│  │ (文集树/大纲   │   │  │                           │  │
│  │  /项目浏览)   │   │  ├───────────────────────────┤  │
│  └────────────────┘   │  │ AICommandBar (折叠状态)    │  │
│                       │  └───────────────────────────┘  │
│                       │  [浮窗: AI 对话 / 审阅 / 插图]  │  ← 右侧浮层，不占网格
├──────────────────────────────────────────────────────────┤
│  StatusBar (精简: 字数 | 保存状态 | 模型)                │
└──────────────────────────────────────────────────────────┘
```

### 关键交互变化

| 变化 | 之前 | 之后 |
|------|------|------|
| **面板数量** | 最多 4 个同时可见（左侧+中间+右侧*2） | 最多 2 个（左侧+中间），AI 为浮层 |
| **右侧面板** | AgentPanel + StylePanel 两个右侧面板 | 合并为右上角浮动抽屉，不占布局网格 |
| **侧栏 tab** | 文件和目录分离 | 目录 / 大纲 / 项目 三个 tab 切换 |
| **新建文档** | 弹窗选合集 | 直接创建到当前合集，标题可双击编辑 |
| **标题编辑** | 需打开蓝图编辑器弹窗 | 双击标题直接编辑 |
| **保存状态** | StatusBar 底部 | 编辑器右上角绿点 + 过渡动画 |
| **焦点模式** | "Exit Focus" 英文按钮 | 右上角中文按钮 + Esc 提示 |
| **工具栏快捷键提示** | 无 | hover 显示快捷键 |
| **AI 面板** | 固定列占据 420px | 浮动弹窗，可拖拽，用完自动收起 |
| **阶段切换** | 静默切换 | 弹出引导对话框 |
| **AIBar 默认状态** | 104px 高度 | 36px 折叠，聚焦展开 |
| **项目面板** | 替代编辑器 | 侧栏 tab，写文章时可快速切换查看 |
| **StartupSplash** | 只有 Logo | 增加引导步骤和快捷键提示 |

## 四、用户典型场景流程对比

### 场景 1：新用户第一次使用

```
之前:
  打开 app → 看到编辑器空白 + Sidebar
  → 点击"新建文档" → 弹窗选合集 → 进编辑器
  → 摸索 Toolbar 各按钮
  → 不知道怎么写第一篇

之后:
  打开 app → 看到 StartupSplash 引导
  → "选择示例文集开始" 或 "创建第一篇"
  → 点击后创建示例文章 + 编辑器自动打开
  → 编辑器内有 "写作提示" ghost text 引导
```

### 场景 2：日常写文章

```
之前:
  点击文集 → 选文章 → 进编辑器
  → 双击标题 → 修改标题（不，需要打开蓝图编辑器）
  → 写内容
  → 底部 AIBar 输入指令

之后:
  点击文集 → 选文章 → 进编辑器
  → 双击标题直接改
  → 写内容
  → 选中文字 → InlineToolbar 弹出快捷操作
  → 底部 AIBar 折叠状态，点击展开
```

### 场景 3：写完后审阅

```
之前:
  写完后切换 phase 到 "reviewing"
  → 打开 AI 面板 → 点击"评估" tab
  → 点击审阅 → 等待 → 看结果
  → 手动修复

之后:
  写完后 → ArticleHeader 显示 "💡 可审阅" 提示
  → 点击 → 触发审阅 → AI 面板浮动弹出
  → 审阅完成显示结果（在当前编辑器上方）
  → 点击"逐项修复" → 逐段优化 → 可接受/拒绝每项
```

### 场景 4：使用 AI 功能

```
之前:
  输入文字到 AIBar → 按 Enter → 等待
  → 结果在 AI 面板展示（必须一直打开右侧面板）
  → 如需查看聊天历史也要在 AI 面板

之后:
  选中文字 → InlineToolbar 弹出快捷选项（润色/改写/翻译）
  → 点击 → 结果以 GhostText 形式直接注入编辑器
  → 按 Tab 接受，Esc 拒绝
  → 如需复杂分析，结果在浮动 AI 面板展示（用完自动关闭）
```
