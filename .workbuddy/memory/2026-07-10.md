## 回收站修复

### Bug 1: 回收站不显示已删除文章
- **根因**：saveTrash 传参数名 `{items}` 但 Rust set_trash 期望 `{trash}`，写入后端静默失败
- **修复**：`{ items }` → `{ trash: items }` + loadTrash 后端返回空时降级 localStorage

### Bug 2: 恢复/删除/清空操作后 UI 不刷新
- **根因**：三个函数都没有 emit("collections-changed")，侧边栏计数和文集树不知道变化
- **修复**：三个函数末尾全部加上 emit

### Bug 3: emptyTrash 只清理列表不清理文件
- **修复**：emptyTrash 改为遍历每个条目执行物理清理后再清空列表

### Bug 4: AI 大纲泄漏思考过程
- **根因**：推理模型的 thinking 链泄漏，且 parseOutline 不识别 ：冒号分隔符
- **修复**：system prompt 增加抑制指令 + 正则支持冒号

### Bug 7: SSE 流中文乱码 — from_utf8_lossy 截断 UTF-8 多字节字符
- **根因**：Rust `openai_chat_stream` 和 `anthropic_chat_stream` 都用了 `String::from_utf8_lossy` 逐 chunk 转字符串。中文字符（如"如"=E5 A6 82）被 TCP 分在两个 chunk 时，第一个 chunk 末尾的 `E5` 被 lossy 替换为 �，永久损毁
- **显示症状**：`如��毁掉`、`你��为` 等乱码
- **修复**：累积原始字节 `Vec<u8>`，只在完整 SSE event 边界（`\n\n`）用 `from_utf8` 解码。event 跨 chunk 截断时保留字节等待下一 chunk
- **根因**：新建文档走 `generateFullArticleStream`，合集规划走 `generateFullArticleWithTools`，前者用户消息缺 tone/targetAudience/targetWordCount，系统提示缺"使用中文"约束
- **修复**：补全三个字段 + 追加语言约束到 system prompt

### Feature: 新建文档分两阶段规划 + 全阶段中文指令
- **buildSystemPrompt** 末尾追加"使用流畅自然的中文"，title/description/outline/tags/writing 所有阶段统一生效
- **generatePlanStream** 新增 `phase` 参数（"full"|"stage1"），stage1 模式只生成标题+描述就停
- **generatePlanStage2** 新函数：根据确认后的标题+描述生成大纲+标签
- **EditorPane** 分两阶段执行 → 后移除（简化回单次生成，排查显示问题）

### Bug 8: 标题/简介不显示
- **根因**：generateTitle 有时返回空字符串 → result.data falsy 检查跳过 → 但即使存进去，空标题 hasTitle = false 也不会渲染
- **fix1**: result.data → 无条件存储（只校验类型）
- **fix2**: generateTitle 在 AI 返回空时截取 inspiration 前 40 字符作 fallback
- **fix3**: generateTitle/Desc 追加抑制指令 + 中文语言约束（不经过 buildSystemPrompt）
- **两阶段规划**（重新实现）：新文档走 stage1 → review-title-desc → 继续 → outline+tags
